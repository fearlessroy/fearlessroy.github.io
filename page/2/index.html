Java中Integer比较的几个注意点

  • == 比较地址，Integer 重写equals()方法，是对值的比较
  • int和integer(无论new否)比，都为true，因为会把Integer自动装箱，然后由于重写了equals 所以比较的是值。
  • 两个都是非new出来的Integer，如果数在-128到127之间，则是true,否则为false，Java在编译Integer i2 = 128的时候,被翻译成-> Integer i = Integer.valueOf(128);JDK本身会对-128到127之间的数进行缓存。
  • Integer 与 new Integer() 不会相等,因为他们在做比较的时候，不会经历拆箱过程，i的引用指向栈，而j指向堆，他们的内存地址不一样，所以为false。

Integer类型判断是否相等不要使用 ==

发表于 2018-03-15 | 分类于 Java

比较数值用equals

  • Integer 是 int 的包装类
  • 包装类与基本类型的转换代码结构是类似的，每一种包装类都有一个静态方法 valueOf(), 接受基本类型，返回引用类型，也都有一个实例方法 xxxValue() 返回对应的基本类型。
  • 将基本类型转换为包装类的过程，称为 “装箱”，而将包装类转换为基本类型的过程，则称为“拆箱”，Java 5 以后引入了自动装箱技术。

  • 自动装箱/拆箱功能是Java编译器提供的能力。

关于Mybatis-Plus字段策略与数据库自动更新时间的一些问题

发表于 2018-03-13 | 分类于 Mybatis

字段策略 0:”忽略判断”,1:”非 NULL 判断”),2:”非空判断”

  • 问题描述：
    当字段策略为 0 “忽略判断” 的时候，如果实体和数据库有字段是需要自动更新的，例如update_time,更新某条记录的时候自动刷新update_time，理论上我们在业务控制的时候是不需要设置update_time的，利用数据库特性就好，但是在该字段策略下，业务逻辑要更新字段（没有主动设置 update_time），这时候因为忽略判断，数据库(我这里是MySql)是不会去更新update_time的，即使这个字段在数据库中是这样定义的：update_time datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE，在这种字段策略下，该字段会被设为null，具体的执行语句时insert into table(….,update_time) values(…,null)。

发表于 2018-02-14 | 分类于 Nginx

openresty快速开始

##简介
Openresty是一个基于Nginx和Lua的高性能Web平台，内部集成了大量精良的Lua库，第三方模块以及大多数依赖项，可以方便搭建超高并发、扩展性极好的Web应用、Web服务和动态网关。

Mac OS X 安装

推荐使用homebrew:
brew install openresty/brew/openresty

安装完后配置环境变量，我用的是zsh，所以：
vim ~/.zshrc
PATH=/usr/local/Cellar/openresty/1.13.6.1/bin:$PATH
保存后source ~/.zshrc使配置生效

第一个web页面

  • 创建工程文件：mkdir ~/first-work
  • 创建日志和配置文件：cd ~/first-work; mkdir logs/ conf/

  • 编写配置文件：

  • 启动Server
    openresty -p ‘pwd’ -c conf/nginx.conf
    这里pwd是你的工程文件地址

  • 打开浏览器访问localhost:8080,便可以看到hello,world

hexo博客绑定腾讯云域名

发表于 2018-01-29 | 分类于 hexo

域名注册

在国内可以通过阿里云或者腾讯云注册域名。
本人域名在腾讯云购入，所以本篇文章以腾讯云为例。
购买注册按流程走即可，各个平台相差应该也不大。

为域名添加解析

修改默认的解析方式，添加 @ 和 www 的DNS记录：

添加加CNAME文件

在本地hexo源码的source目录创建一个CNAME文件，文件内容是域名，我的域名是 fearlessroy.net。
这里不推荐直接在Github仓库创建CNAME文件，因为在下次部署内容之前忘记同步文件的话会无法访问我们的域名。
hexo g & hexo d之后，Github上便会有我们的文件。

完成

等待一会(域名解析生效的时间，在添加域名解析时会有提示)，访问注册的域名，便会出现和原先的二级域名一样的页面，至此博客绑定域名成功。
